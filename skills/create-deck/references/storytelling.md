# Storytelling

Use this reference before Phase 1 planning.

## Objective

Build decks that read like a recommendation with proof, not like topic notes.

## Pyramid Principle

Organize the story in this order:

1. Answer first.
2. Group the supporting arguments.
3. Present the evidence under each argument.

In deck terms:

- opener: what this deck is about and why it matters
- core slides: the arguments that support the recommendation
- proof slides: evidence, comparisons, examples, charts, or source-backed facts
- closer: the action or takeaway

## Action Titles

Every content slide title should express the conclusion of the slide.

Good:

- "Retention gains come mostly from onboarding improvements"
- "The migration cost pays back within two quarters"

Weak:

- "Retention"
- "Migration Analysis"

Rule: if the audience reads only the titles, they should still understand the story.

## Storyline Checklist

- Start from the answer, not the background.
- Make each slide earn its place in the argument.
- Separate claims from proof.
- Keep one message per slide.
- Use evidence that matches the claim.
- End with a recommendation, implication, or next action.

## Default Deck Arc

For most business decks:

1. Title / framing
2. Situation or problem
3. Core insight or recommendation
4. Supporting proof
5. Implication / next step

Expand only when the topic genuinely needs more argument layers.
